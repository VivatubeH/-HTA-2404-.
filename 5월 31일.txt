continue 문

for문 내에서의 사용 : continue가 실행되면 수행문을 실행하지 않고, 
증감식으로 이동해서 다음 번 반복을 수행한다.

break나 continue를 사용하면 코드의 가독성을 저하시킬 수 있음.
[ 꼭 필요한 경우에만 사용하기. ]

중첩된 for문 : for문 안에 for문을 쓰는 것. 
[ 중첩된 for문은 1번까지만 중첩하는 게 좋음 ]
[ outer 수행문 : 자기 횟수만큼, inner 수행문 : outer 횟수 x 자기 횟수 ]
- 반복문 내에서 또 다른 반복을 수행할 수 있게 함.
[ 구구단 출력처럼 출력하는 게 가능하다. ]
[ \t는 탭문자임. - 스페이스 여러개와 동일한 너비로 글자를 띄우는데 사용 ]

규칙을 찾아서 못 찾으면 그냥 다른 변수를 하나 추가하기.
반복문에서 그냥 변수 활용 안하고 그냥 사용해도 된다.

코드를 최대한 간략하게 쓸 수 있는 방법 찾기.
[ for문에서 초기값과 조건식을 어떻게 주느냐에 따라 반복횟수를 조절할 수 있음 ]

for문으로 무한 루프 만드는 법
for(;;)와 같은 방식으로 작성하면 무한 루프가 된다.

for문의 초기식에서 선언한 변수는 블록 밖에서는 사용할 수 없다.

자바에서는 닫는 중괄호 ( 블록이 끝나면 )를 만나면 변수가 즉시 소멸된다.

while문은 개발자가 반복의 횟수를 지정하는 것이 아니라,
반복 작업의 대상이 되는 객체가 반복을 지속할 지 여부를 알려줌.

자바에서는 배열보다는 컬렉션을 많이 사용함 [ 코테에서는 배열 사용할 듯? ]

배열 - 배열 객체는 한 번에 여러 개의 데이터를 저장할 수 있음.
[ 인덱스는 0부터 자동으로 부여된다. ] 
[ 1. 무엇을 담을지 2. 크기를 얼마로 잡을지를 결정해줘야 함 ]

자바에서는 기본형 8가지만 값을 담는 변수다.
[ 8가지를 빼고는 전부 다 주소를 담는다. ]
[ 즉, int[] 이렇게 생겨도 그냥 주소를 담는다. ]

자바에서 객체가 아닌 것
- 기본 자료형, 연산자 

new int[5]; 에서 new의 역할
1) 배열 객체를 생성한다.
2) 생성된 객체의 주소값을 변수에 대입한다.

객체 : 메모리의 heap 영역에 생성되는 속성과 기능을 갖는 것. 

* 배열의 생성목적
1. 여러 개의 값을 하나의 이름으로 관리. 
2. 인덱스를 사용해서 쉽게 값을 저장/조회.

자바에서 배열의 크기는 한 번 만들어지면 변경 불가함. [ 변형 불가함 ]

자바의 배열은 동일한 타입의 값만 저장할 수 있다.

참조를 더 이상 못하게 하고 싶을 때 null을 사용한다.
자바에서는 메모리에 직접 접근은 불가능하다.

아무도 참조하고 있지 않은 객체는 가비지컬렉터의 청소 대상이 된다. [ 사라진다. ]
[ 자바는 한 번 연결이 끊어진 객체와는 절대로 연결이 불가능함. ]

참조변수는 타입에 관계 없이 null이라는 값을 가질 수 있음.
[ 단, 자기와 맞는 객체의 주소값만 담을 수 있음. 다른 객체의 주소값을 담을 수 없음 ]
[ null 은 객체를 참조하고 있지 않은 참조변수가 갖는 값 ]

모든 참조변수는 주소값을 가지는 순간, 해당 객체와 연결된다.

바라보는 객체가 하나라도 있으면 가비지가 아니다.

배열을 생성하는 방법
1. 기본값으로 채워지는 배열
2. 미리 설정한 값으로 채워지는 배열

자바는 배열의 크기를 지정하지 않고는 만들 수 없다. [ 크기를 정하거나, 값을 정하거나 ]

배열, 컬렉션 전용의 for문은 세미콜론이 아닌 콜론을 사용한다.
